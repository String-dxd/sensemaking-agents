## Phases & milestones (gate each phase before the next)

1. **Prove it lives** (001–003): placeholder character with a drawn face that
   blinks and watches the cursor, ears/tail that lag/overshoot/settle, visible
   breathing. *Go/no-go gate for the whole project.*
2. **A real character** (004–006): dog and bird assembled from authored parts,
   palette-recolored, Pokopia-grade shading, still alive.
3. **Motion & identity** (007–008): full Play Mode (idle/walk/run/sit/talk/
   gestures + soak test); dressed characters whose clothing moves.
4. **Authoring power** (009, 010, 012): unique silhouettes beyond sliders,
   relightable stage, roster persistence. (009 ∥ 010 ∥ 012 are parallelizable.)
5. **Ship the roster** (011): exported `.companion.glb` plays with full
   secondary motion in a three@0.149-compatible runtime package.

## Standing constraints (from 000 — repeated because they get violated)

- `character-studio/` is an **isolated pnpm root** (bird-builder pattern).
  Never add it to root `pnpm-workspace.yaml`; never touch root `overrides`.
- `src/core/**` never imports React (mechanically enforced by a plan-001 test).
- Frame order: animation → physics(springs) → procedural → render. Never reorder.
- Face planes are always unlit. Spring-bone chains are never keyframed.
- No `Math.random` in core (inject seeded RNG). No retargeting anywhere.
- Fixed mesh topology (no dyntopo). meshopt+KTX2, not Draco. GLB self-sufficient
  for naive viewers; `SEN_companion` extension carries the "alive" data.
