## Why this matters

After plan 030 (retina DPR + 512-segment tessellation), the maintainer sent a
second screenshot: the coastline still reads as a staircase — long flat runs
of 2–4 grid cells with hard 1-cell steps. This is NOT a rendering artifact
anymore. The saved island's tier grid is 64×64 over a 24-unit world (cell =
0.375 units), and the runtime field smoothing cannot fix multi-cell steps:
`sampleTierField` mixes only `BLUR_MIX = 0.4` of the blurred field with 60 %
of the raw binary grid — by design, to preserve thin authored features (see
the WHY comment on `sampleTierField`,
`src/engine/student-space/Game/State/islandSpecCore/terrainGrid.ts:194-205`).
The macro staircase lives in the DATA, so the fix the maintainer asked for
("update the saved island so it's smooth and curved") is a one-time data
transformation:

**Resample the saved island 64×64 → 128×128, thresholding a strongly-blurred
continuous field** so the new, finer binary grid traces a genuinely curved
coastline. The runtime smoothing then rounds the remaining 0.1875-unit steps
easily. The one-time resample may use a much stronger blur than the runtime
mix because it is an explicit, maintainer-requested artistic transformation —
the thin-feature invariant protects live editing, not this offline bake.

Everything downstream is already resolution-agnostic (verified at `32e2c92`):
the spec validator accepts any positive cols/rows (`specIO.ts:52-79`), the
grid codec encodes per-row digit strings of arbitrary length, terrain
geometry / shore field / grass / object placement all derive from
`spec.grid.cols`, shore distances are in WORLD units (`shoreField.ts:13-16`),
and plan 030's `SEGMENTS = 512` is 4 segments per cell at 128 — the same
ratio the geometry comment originally required at 64. Only ONE thing needs a
code change: grass density is per-cell (`BLADES_PER_CELL = 64`,
`grassField.ts:35`), so a 4× cell count would 4× the ~16k blades; compensate
by dropping the constant to 16 (same blades per world area).

## Steps

### Step 1: Write `scripts/resample-island-grid.ts`

Model the header/style on `scripts/sync-island-spec.ts` (tsx, node:fs, engine
imports). The script:

1. Reads and validates `island-editor/saves/island.json` via the ENGINE
   `validateSpecObject` (import from
   `../src/engine/student-space/Game/State/islandSpecCore/specIO.ts`).
2. If `spec.grid.cols !== 64`, print "already resampled" and exit 0
   (idempotence guard).
3. Computes a script-local strongly-blurred field over the OLD grid — do NOT
   modify core modules:
   - `RESAMPLE_BLUR_PASSES = 3` applications of the same 3×3 tent kernel as
     `blurTiers` (`(1 2 1 / 2 4 2 / 1 2 1)/16`, out-of-bounds = tier 0).
   - A script-local smooth-bilinear sampler identical to the core's
     (`fu = fu*fu*(3-2*fu)` on both fractions, cell-center space, clamped).
   - Continuous field `t(x, z) = raw(x,z) * (1 - M) + blurred(x,z) * M` with
     `RESAMPLE_MIX = 0.75`. Comment WHY this may exceed the runtime
     `BLUR_MIX = 0.4`: one-time maintainer-requested artistic bake; the
     runtime mix's thin-feature invariant protects live editing, not this.
4. Builds the new grid, 128×128 (`cols*2 × rows*2`):
   - `tiers[r'*128 + c'] = clamp(Math.round(t(center of new cell)), 0, MAX_TIER)`
     with new-cell centers from the core `cellCenter` semantics
     (`-worldSize/2 + (c'+0.5) * worldSize/128`).
   - `surface[...]` = nearest-neighbor from the old cell containing the new
     center (`floor(c'/2)`, `floor(r'/2)`).
5. Remaps objects: `c' = c*2`, `r' = r*2` (center shifts ≤ ~0.09 world units).
   Then for EVERY object (including the character): if
   `tiers[r'*128 + c'] < 1` (not land), search Chebyshev radius 1..3 for the
   nearest new cell with tier ≥ 1 and relocate; if none exists within radius
   3, STOP (print the object id and exit non-zero).
6. Prints stats and enforces bounds (exit non-zero if violated):
   - land-cell fraction (tier ≥ 1): old vs new must differ by < 15 % relative;
   - per-tier cell counts old×4 vs new (informational);
   - max tier present must be unchanged;
   - number of relocated objects (informational).
7. Writes `serializeSpec(newSpec) + '\n'` to `island-editor/saves/island.json`.

**Verify**: `pnpm exec tsx scripts/resample-island-grid.ts` → exit 0, stats
within bounds (paste them in your report). Run it a second time → prints
"already resampled", exit 0, file unchanged (`git diff --stat` shows the save
modified once).
