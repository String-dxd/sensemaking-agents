## The core reframing

A first read of the code suggests the app is a *one-shot* reflection tool
(`mirror.prompt.md` bans questions). It is not. The live voice conversation runs
on a **different** prompt (`mirror-realtime-live.prompt.md`) that is already
multi-turn and conversational. The "no questions" rule governs only the silent
post-conversation JSON notes pass. So this work is **alignment of what exists with
the spec**, not a new interaction model. Several spec "gaps" also turned out to be
already built (see 002).

## What each plan does (one line)

- **001** — Adds Ming Liang as a 5th seed student (the PDF's 9 transcripts) so the
  canonical demo runs end-to-end. Bumps a seed-count test ceiling.
- **002** — Adds the ECG career-fair teacher prompt as a seed letter. The
  letter→capture mechanic **already exists**; this is pure content.
- **003** — Rewrites onboarding copy (values only, flow kept) to the PDF's
  voice-first "tap the mic → three things that connect → looks like you" register.
- **004** — Makes the live conversation Singlish-aware, adds a spoken
  readback-for-confirmation move, a GROW coaching mode, and a teacher-prompt
  opener rule. Leaves the English JSON notes pass untouched.

## Decisions the maintainer should confirm (baked defaults in the plans)

The plans proceed on these recommended defaults; override before executing if you
disagree:

1. **Naming** (000 Decision 1): keep `Mirror` as internal id, companion stays
   user-named, do **not** introduce "Mei" as a narrator name (collides with a demo
   student). A hard product-facing rename to "Kira" would be a separate plan.
2. **`[feeds]` tags** (000 Decision 2): do **not** extend the VIPS taxonomy; the
   seven feed words map onto the existing four dimensions + `context_type` +
   Trajectory. Seed data uses existing canonical ids only.
3. **Onboarding** (003): revise copy, **keep** the egg/bird-hatching ceremony.
   Collapsing to literally three screens (deleting choreography) is a bigger,
   sign-off-gated change — not in this suite.
4. **Singlish split** (004): the *spoken* conversation becomes register-matching;
   the *stored* VIPS notes stay clean English. Intentional.

## Findings considered and rejected / de-scoped

- **"Build a teacher-prompt mechanic"** — rejected as a build task: it already
  exists (`LettersSheet.tsx` opens an `ask` capture from a letter's `prompt`
  field). Reduced to content only (002). A stale comment in `TeacherLetters.js`
  claiming it's unbuilt is corrected in 002.
- **"Rewrite `mirror.prompt.md` to allow questions"** — rejected: that file is the
  JSON structured-output pass, not the conversation; its rules are correct.
- **"Extend the VIPS taxonomy to add relationships/identity/choices/strengths"** —
  rejected: expressible in the existing schema (see 000 Decision 2). Revisit only
  if the product deliberately shifts to a richer stored taxonomy.
- **"Product-wide rename Mirror → Kira"** — de-scoped: high churn, low user value;
  internal naming is invisible to students. Separate plan if ever wanted.
- **Transcription-language change for code-switching** — de-scoped to a measured
  follow-up experiment inside 004 (prompt-level register matching is the safe win).
