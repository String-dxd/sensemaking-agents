## Decisions in force (see 000 for detail)

1. **Naming**: student = Ming Liang; bird default name = "Mei", editable;
   "Kira" in transcripts = companion-name stand-in; copy always uses
   `{companionName}`; `Mirror` stays the internal agent id.
2. **Taxonomy**: `[feeds]` tags map onto the existing 4 VIPS dimensions +
   `context_type` + Trajectory — no taxonomy changes.
3. **Demo identity**: keep student id `demo-a`, replace content; `demo-b..d`
   untouched.
4. **Moods/dates**: seeded via `mood:*` tags (never `embarrassed` — id
   mismatch across registries); spec timeline compressed Jan→current week.

## Findings considered and rejected / de-scoped

- **Add Ming Liang as a 5th student** (001 v1) — superseded: the demo signs in
  as `demo-a`, so a 5th student would never be seen; content replacement wins.
- **Collapse onboarding to literally 3 screens** — rejected: deletes the
  hatching ceremony choreography without product sign-off; the script maps
  cleanly onto the existing narrator beats (003).
- **Hardcode "Mei"/"Kira" into copy** — rejected: companion stays user-named;
  "Mei" is only the editable default (maintainer decision 2026-07-13).
- **Rewrite `mirror.prompt.md` to allow questions** — rejected: it governs the
  post-hoc JSON pass, not the conversation.
- **Extend the VIPS taxonomy** — rejected: expressible in the existing schema
  (see 000 Decision 2).
- **Build a teacher-prompt mechanic** — rejected: already exists
  (`LettersSheet.tsx` opens an `ask` capture); 002 is content-only.
- **Product-wide rename Mirror → Kira** — de-scoped: high churn, invisible to
  students.
- **Transcription-language change for code-switching** — de-scoped to a
  measured follow-up inside the 004 rewrite.
