## Architecture facts that frame the work

1. **The exported terrain spec is consumed by nothing.** `grep -rn "IslandSpec\|evaluateHeight" src/`
   (repo root) returns zero hits — the product engine (`src/engine/student-space/Game/State/Island.js`)
   still hard-codes terrain (`radius=5.0`, `silhouetteAt`, `heightAt`). The editor's seed even *copies*
   that silhouette (`islandSpec.ts:169`). This is **deferred-by-design** per the `2026-06-15-000`
   overview ("island shape/terrain editing (deferred v2)… collides with the ambient-visual rebuild").
2. **PR #74 made object placement & species data-driven but removed the in-app `#editor`.** The engine
   now boots from committed `defaultIslandLayout.json` (31 objects) + `defaultSpeciesPalette.json`; the
   standalone editor authors **terrain only**, not those layers.
3. **The standalone editor is an isolated pnpm workspace** — excluded from the repo's `pnpm check`
   gate (which is scoped to `src test`).

## Remaining works / direction (NOT built — maintainer's call)

These are real gaps the audit surfaced but did **not** turn into build plans (per the requester's
choices and honest scoping):

- **REMAIN-01 — Wire the terrain spec into the engine.** The editor's whole purpose (author terrain
  that ships) is unfulfilled today. Deferred-by-design (collides with the ambient-visual rebuild), and
  the requester chose a *design spike* over building. The cheapest first move is a consumption-contract
  investigation (how would `Island.js` read an `IslandSpec` heightfield, at what cost). **Effort L; partly
  blocked.** Plan 001's binding section documents that in-product agent editing is blocked on this.
- **REMAIN-02 — No GUI for placement/species.** Since #74, moving a tree means hand-editing
  `defaultIslandLayout.json`; species colors are hand-edited `defaultSpeciesPalette.json`. The
  standalone editor could grow a placement/palette mode (it already renders terrain in r3f). **Effort L.**
- **QUAL-05 — Relief is a dense 192² (36,864-float) grid** serialized whole on every autosave and
  export, almost always mostly-zero — and awkward for an agent to author/diff (tracked in 001's open
  questions). A sparse/RLE encoding or a lower default resolution would help once agent-editing is real.
  **Effort M; deferred** (changes the spec format → versioning).

## Findings considered and rejected (so they aren't re-audited)

- **QUAL-03 — Seed silhouette duplicated** between `islandSpec.ts:171` and `Island.js:31`. Real but
  isolated-by-design (keeps the editor free of the three@0.149 boundary); risk is silent drift if the
  engine silhouette is tuned. **Verdict: maintenance note, not a plan** — captured in 001/004 notes.
- **QUAL-06 — `INITIAL.relief` is module-level and mutated in place by the brush;** a StrictMode/HMR
  remount could reuse the mutated array as the ref seed. **Verdict: LOW confidence it manifests**
  (reset/import/reload all replace the ref; reload re-reads localStorage). Not worth a plan; revisit
  only if odd brush-state-on-reload is observed.
- **EDIT-04 — No orthographic/elevation camera presets** beyond "Top view". **Verdict: low value for a
  dev tool**; "Top view" plus orbit suffices.
- **AGENT live binding (in-product MCP/managed-agent tool)** — the requester explicitly chose
  "design spike only," and it's blocked on REMAIN-01 anyway. **Verdict: out of scope now**; revisit
  after the engine consumes a terrain spec.
