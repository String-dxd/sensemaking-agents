## Root-cause summary (for reviewers)

1. **Walk didn't articulate** — the Studio-mode debug "walk circle"
   (`bodyMover.ts:45-51`) slides the root only; the fully-authored,
   correctly-wired walk clip runs exclusively in Play mode. → Plan 001.
2. **Floating mouth / face parallax** — face parts are hover planes on an
   analytic sphere (`headRadius × 1.07` + per-muzzle `mouthOffset` up to
   0.14 m radial push), a technique that cannot hug real geometry. AC draws
   faces in the head's own UVs. → Plan 002.
3. **Half-cut arms / visible interior / fins** — bodies are overlapping
   shells with rest-pose-only SDF fillets (`bodies.py`); any bone rotation,
   morph, or bone-scale tears the junction; front-side culling exposes the
   hollow interior. AC bodies are one welded mesh. → Plan 003.

## Known issues surfaced during execution (not yet planned)

- **UV-wrap seam stripe on the back** (observed post-007 in the browser,
  likely pre-existing): the head/torso islands' wrap edges sit adjacent to
  OTHER islands in mask-image space, so `rasterize_mask`'s box blur bleeds
  neighboring-island channels across island boundaries — visible as a thin
  light stripe down the back centerline under the toon shader. Fix
  candidate: pad each island's rasterization with edge-dilated gutters
  before blurring (small `meshkit.py` change).
- **AC proportion calibration reference** (plan 007): the durable numbers
  live in the plan's benchmark table; the AC render PNGs used for visual
  acceptance were session-temporary and NOT committed (Nintendo assets).
- **Part-GLB export nondeterminism**: the 8 part GLBs under
  `src/assets/anatomy/parts/` byte-drift on EVERY `pnpm gen:assets` run under
  Blender 5.1.2, including on the untouched pre-weld baseline (observed by
  the plan-003 executor, who reverted them each run — they are not in any
  003 commit). Likely Blender-version export nondeterminism. Harmless at
  runtime but makes asset diffs noisy; candidate for a small follow-up.

## Findings considered and rejected

- **"Foot IK is inert during walk" (audit finding MOTION-02)**: RESOLVED BY
  MEASUREMENT (plan 004, `test/core/motion/gaitSoak.test.ts`). Both static
  readings were partially wrong. Measured: idle engagement 1.0 / drift
  0.09 mm (healthy); walk engagement 0.246 with 4.5 mm max drift while
  pinned (the anti-skate layer DOES work at walk); run engagement 0.000 —
  the gate is fully inert at run because the authored clips' stance-foot
  speed is only correct as a cycle AVERAGE (bottoms at ≈0.894/≈1.798 m/s in
  place, matching the constants) but the instantaneous contact-window
  residual at run never drops below ≈0.86 m/s, over the 0.4 m/s gate.
  Steady-state (straight-line control reproduced it) → the fix is clip-side
  (flatten the contact-phase foot-speed profile in
  `scripts/blender/clips.py`), tracked as plan 006. The measured baseline is
  pinned; a clip fix should flip walk/run pins to healthy bounds
  (ratio > 0.5, drift < 0.02).
- **Clip loading / bone-name binding failures**: traced and cleared — no
  silent failures; dotted-bone-name sanitization is already handled
  (`assemble.ts:78-83`); the clip GLB verified complete (11 clips, walk has
  22 channels incl. arms/legs).
- **Dual root ownership (bodyMover vs locomotion during Play)**: real but
  latent (needs a debug toggle active on Play entry); folded into Plan 001
  Step 5 rather than a standalone plan.
- **Pokopia as a second benchmark**: dropped — shipped 2026-03, no teardowns
  exist, and reviews indicate a different rendering lineage (voxel world,
  Ditto player character). AC:NH is the actionable benchmark.
